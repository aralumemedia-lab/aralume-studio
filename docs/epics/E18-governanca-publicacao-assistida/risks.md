# E18 - Risks

| Risk | Evidence | Impact | Mitigation |
| --- | --- | --- | --- |
| Governance can drift back into read-only inspection | The acceptance matrix still marks V1-11..V1-14 as not proven. | The flow may look complete while decisions remain unproven. | Require mutating actions and reload proof for each story. |
| Quality and compliance frontend surfaces are currently read-only or partial | `src/routes/approvals.tsx` and `src/routes/compliance.tsx` primarily query existing records; the bundle requires operator-visible gate mutations. | V1-11 and V1-12 can remain NOT PROVEN despite working backend rules. | Make Sprint 19 a vertical slice with real API mutations, blocked cases and browser evidence. |
| Governance/publication audit entries lack requestId propagation | Current governance and publication services record audit metadata without receiving the HTTP requestId; response envelopes alone do not correlate mutations. | Evidence cannot prove mutation-level traceability. | Add requestId propagation and regression assertions in Sprint 19 and Sprint 20. |
| Approval creation is not exposed by the current frontend path | `/approvals` has decision actions but does not currently prove creation of the approval record in the operator flow. | H18.3 may depend on seeded state rather than a complete frontend journey. | Include approval creation or controlled selection in the Sprint 19 DoR and E2E. |
| Publication surface contains external provider actions | `/publications` includes YouTube connection and upload paths in addition to target/job readiness. | A sprint can silently expand from assisted preparation into external publication. | Keep H18.4 limited to readiness or draft package and assert no external call. |
| E18 in one sprint would couple unproven gates | H18.4 depends on H18.1-H18.3 and has a different no-auto-send boundary. | Mixed E2E and unclear gates can make the epic appear complete prematurely. | Split into Sprint 19 governance gates and Sprint 20 assisted publication. |
| Publication readiness may silently imply auto-send | The spec explicitly forbids external auto-send. | The plan could accidentally cross into release behavior. | Keep the no-auto-send rule in scope and test it explicitly. |
| Process-local persistence is not restart durable | The current baseline does not add a database for this epic. | Reload after process restart remains unproven. | Accept only same-process reload persistence in this epic and document the limit. |
| Visual regression can hide state loss | Governance surfaces are dense and can overflow with long values. | A layout defect can hide missing data. | Require screenshots at all mandated viewports and explicit empty/error states. |
